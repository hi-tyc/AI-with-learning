# StudyBuddy AI 伴学系统 — 使用指南

> 面向初中生的 AI 学习辅助工具，支持拍照识别、AI 分步解题、题库管理与错题本。

---

## 一、系统概述

StudyBuddy 是一个全栈 Web 应用，由 **FastAPI 后端** + **Vue 3 前端** 组成：

- **后端**：FastAPI（Python），提供 RESTful API + SSE 流式解题
- **前端**：Vue 3 + Element Plus + Vite，现代化单页应用
- **数据存储**：本地 JSON 文件（无需数据库，零配置运行）
- **AI 能力**：支持 DeepSeek 和 Kimi 双模型，按题目类型自动分流

### 核心功能

| 功能 | 说明 |
|------|------|
| 📸 拍照传题 | 支持 JPG/PNG 上传，自动压缩，Magic Number 校验 |
| ✏️ 手动录入 | 直接粘贴题目文本，快速入库 |
| 🧠 AI 解题 | SSE 流式输出，实时显示解题过程 |
| 📚 题库浏览 | 六级路径树（学科→考试→资料→学校→大题→小题），关键词搜索 |
| ❌ 错题本 | 一键标记/移除错题，方便复习 |
| ⚙️ 管理模式 | 路径重命名、批量编辑、共享题库 |
| 🔧 系统设置 | 灵活配置 API Key、模型、图片压缩、超时重试 |

## 二、环境准备

### 1. 前置要求

| 依赖 | 最低版本 | 用途 |
|------|----------|------|
| Python | 3.10+ | 后端运行 |
| Node.js | 18+ | 前端开发 |
| uv（可选） | 最新版 | 快速 Python 环境管理 |

### 2. 安装依赖

**后端（Python）**：

```bash
cd backend
# 方式一：使用 uv（推荐）
uv venv venv
uv pip install -r requirements.txt --python venv/Scripts/python.exe

# 方式二：使用 pip
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

**前端（Node.js）**：

```bash
cd frontend
npm install
```

## 三、启动系统

### 一键启动（推荐）

在项目根目录下双击运行：

```bash
start.bat
```

该脚本会同时启动：
- 后端：`http://localhost:6000`
- 前端：`http://localhost:5173`

### 手动启动

**终端 1 — 启动后端**：

```bash
cd backend
venv\Scripts\python.exe -m uvicorn main:app --reload --port 6000
```

**终端 2 — 启动前端**：

```bash
cd frontend
npm run dev
```

### 访问地址

打开浏览器访问：`http://localhost:5173`

## 四、首次使用流程

### 1. 注册 / 登录

- 打开 `http://localhost:5173`
- 输入任意用户名（1-20 位），点击「登录 / 注册」
- 首次登录自动创建账户和默认配置

### 2. 配置 API Key（必须！）

**解题功能需要 AI 模型支持，必须配置至少一个 API Key：**

1. 登录后点击顶部导航栏 **「设置」→ 系统设置**
2. 配置以下任一（或全部）：

| 配置项 | 获取方式 | 说明 |
|--------|----------|------|
| DeepSeek API Key | [DeepSeek 官网](https://platform.deepseek.com) | 性价比高，推荐日常使用 |
| Kimi API Key | [Moonshot 开放平台](https://platform.moonshot.cn) | 支持图片识别，几何题必选 |

3. 点击「测试连接」按钮验证 Key 是否有效
4. 调整其他偏好设置（见下方「系统设置详解」）
5. 点击「保存设置」

> 💡 **提示**：建议同时配置两个 Key，系统会自动按题目类型选择最优模型。

## 五、核心功能详解

### 1. 📸 拍照上传题目

**步骤：**
1. 进入「解题」页面
2. 左侧「拍照 / 传题」区域，拖拽或点击上传图片
3. 系统会：
   - 校验文件类型（仅 JPG/PNG，通过 Magic Number 识别）
   - 检查文件大小（超出限制会拒绝）
   - 自动压缩图片（按设置的长边/质量参数）
   - 生成 Base64（供后续 AI 识别使用）
4. 右侧「确认题目信息」卡片会显示：
   - 压缩前后大小对比
   - 图片预览
   - 可编辑的识别内容（如已有）
5. 填写学科、考试/章节、资料来源、学校、大题号、小题号、知识点
6. 勾选「标记为错题」或「共享到题库」（可选）
7. 点击「确认入库」

> ⚠️ **注意**：上传图片需要配置 **Kimi API Key** 才能进行图片识别。

### 2. ✏️ 手动录入题目

适合电脑做题、网页复制等场景：

1. 进入「解题」页面
2. 右侧「手动录入」卡片
3. 选择学科、粘贴题目内容、填写知识点
4. 点击「录入题库」

### 3. 🧠 AI 解题

**步骤：**
1. 在「解题」页面下方的「解题列表」中，找到想解的题目
2. 选择解题策略：
   - **自动**：系统判断（有图→Kimi，无图→DeepSeek）
   - **几何**：强制使用 Kimi（支持图片理解）
   - **无几何**：强制使用 DeepSeek（速度快、成本低）
3. 点击「解题」按钮
4. 观看 **SSE 流式输出**——答案逐字实时显示，无需等待
5. 解题完成后，结果自动保存到该题目下

**解题界面说明：**
- 题目会附带你的年级、学校信息，AI 会据此调整讲解深度
- 公式以 LaTeX 格式呈现（如 `\(x^2 + 2x + 1 = 0\)`）
- 讲解分步进行，不跳步，适合初中生理解

### 4. 📚 题库浏览

进入「题库」页面：

**左侧 — 路径树**：
- 按「学科 → 考试/章节 → 资料来源 → 学校 → 大题 → 小题」六级树形展示
- 点击任意节点，右侧自动筛选该路径下的题目

**右侧 — 题目列表**：
- 关键词搜索（支持内容、知识点模糊匹配）
- 学科筛选下拉框
- 错题筛选（是/否）
- 分页浏览
- 点击「详情」进入题目详情页
- 点击「删除」移除题目（会同步从错题本移除）

### 5. 📋 题目详情页

点击题目进入详情：
- 查看完整题目信息（学科、来源、学校、知识点等）
- 查看/重新触发 AI 解答
- 点击「加入错题本」/「移出错题本」

### 6. ❌ 错题本

进入「错题本」页面：
- 集中展示所有标记为错题的题目
- 显示加入时间、学科、知识点
- 支持「移出错题本」和「查看详情」
- 空状态时提示前往解题页面添加

> 💡 **复习建议**：定期回顾错题本，重点攻克高频出错知识点。

### 7. ⚙️ 管理模式

进入「管理」页面：

**路径管理**：
- 输入旧路径和新路径（如 `数学/期末` → `数学/期中`）
- 一键批量重命名该路径下的所有题目

**题目编辑**：
- 分页展示所有题目
- 支持直接编辑题目内容并保存
- 支持删除题目

**共享题库**：
- 查看其他用户共享的题目（如班级共建题库）

### 8. 🔧 系统设置详解

进入「设置 → 系统设置」：

#### AI 模型配置

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| DeepSeek API Key | 空 | sk- 开头 |
| DeepSeek 模型 | flash | flash（快）/ pro（强） |
| Kimi API 类型 | platform | platform（平台版）/ code（Code 版） |
| Kimi API Key | 空 | Bearer Token |
| Kimi 端点 | `https://api.moonshot.cn` | 国内用户默认即可 |

#### 用量控制

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 每日花费上限 | 10 元 | 防止意外高消费，超限自动拒绝 |

#### 图片处理

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 图片最大大小 | 5 MB | 超过此大小拒绝上传 |
| 压缩质量 | 85% | JPEG 压缩质量，越高越清晰 |
| 压缩后最大长边 | 1200 px | 超过则等比缩放 |

> 💡 压缩流程：Pillow → OpenCV → imageio，三级自动降级，确保任何环境都能压缩。

#### 超时与重试

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| Kimi 超时 | 30 秒 | 图片识别需更长时间 |
| DeepSeek 超时 | 60 秒 | 长文本推理需更久 |
| 重试次数 | 2 次 | API 失败自动重试 |
| 重试间隔 | 3 秒 | 每次重试间隔 |

### 9. 👤 个人资料

进入「设置 → 个人资料」：
- 修改年级（六/七/八/九年级）
- 修改学校名称
- 这些信息会传递给 AI，影响讲解风格

## 六、数据存储说明

所有数据存储在项目目录的 `backend/data/` 文件夹下，纯 JSON 格式：

```
backend/data/
├── users/                  # 用户数据
│   ├── {username}.json     # 用户资料
│   ├── {username}_config.json  # 系统配置
│   ├── {username}_problems.json # 个人题库
│   └── {username}_wrong.json    # 错题本 ID 列表
├── uploads/                # 上传的图片
│   ├── {uuid}_raw.jpg      # 原始图片
│   └── {uuid}_compressed.jpg  # 压缩后图片
├── shared/                 # 共享题库
│   └── shared_problems.json
└── memory/                 # 记忆系统
    └── {username}_daily.json
```

**备份与迁移**：
- 直接复制 `backend/data/` 文件夹即可完整备份
- 更换电脑时，复制该文件夹到新环境即可恢复所有数据

## 七、常见问题

### Q1：登录后页面空白或报错？
- 确保后端服务已启动（`http://localhost:6000/health` 应返回 `{"status":"ok"}`）
- 检查前端 `vite.config.js` 中的代理配置是否正确

### Q2：点击「解题」无反应？
- **必须先配置 API Key！** 进入系统设置填写 DeepSeek 或 Kimi 的 Key
- 检查浏览器控制台（F12）是否有网络错误

### Q3：上传图片失败？
- 检查文件是否为 JPG 或 PNG（系统通过文件头 Magic Number 校验，不依赖扩展名）
- 检查图片是否超过「系统设置」中设定的最大大小
- 检查后端是否有 Pillow / OpenCV 等图片处理库（已包含在 requirements.txt）

### Q4：AI 回答卡顿或超时？
- 在系统设置中增加「超时」时间（DeepSeek 建议 60-120 秒）
- 检查网络连接
- 查看模型是否被限流（API 服务商侧问题）

### Q5：如何清空所有数据？
- 直接删除 `backend/data/` 目录下的文件即可
- 用户数据按用户名隔离，删除其他用户文件不会影响当前用户

### Q6：如何共享题目给同学？
- 入库时勾选「共享到题库」
- 同学在自己的「管理 → 共享题库」中查看
