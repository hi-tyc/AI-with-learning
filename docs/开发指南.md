\# StudyBuddy - AI 伴读系统（精简开发任务书）



\## 一、项目定位



为 1-4 人私有化部署的初中学习辅助系统。核心能力：拍题/PDF → AI 识别 → 模型分流（代数走 DeepSeek，几何走 Kimi）→ 解题 → 按六级路径存储为 JSON → 精准检索。



\*\*关键特性\*\*：

\- 这是一个中等复杂度的全栈项目，不要只写几个文件敷衍。前后端分离，模块众多，请按功能合理拆分成足够多的文件，每个模块独立维护。

\- 所有配置（API Key、模型选择、每日上限、超时时间、图片压缩率）均通过前端界面设置，存到 JSON 文件，无需改代码或环境变量。

\- 提供 Windows 一键启动命令，双击同时跑前后端。



\*\*开发环境\*\*：Windows 10/11，本地运行。

\*\*部署环境\*\*：Linux 服务器（后期迁移）。

\*\*开发端口\*\*：后端 `http://localhost:6000`，前端 `http://localhost:5173`。

\*\*部署时\*\*：后端端口改为 `8000`，前端 `build` 后作为静态文件部署。

\## 三、核心功能模块（你设计实现细节，但功能必须全覆盖）



\### 3.1 系统配置管理（核心）



\*\*所有配置均通过前端界面 `/settings/system` 完成，每个用户独立保存，存到 JSON 文件。\*\*



前端设置页面包含：

\- DeepSeek API Key + 模型选择（flash/pro）+ 测试连接

\- Kimi API 类型（平台/Code）+ API Key + 测试连接

\- 每日 API 花费上限、单次图片最大尺寸、图片压缩率（%）、图片压缩后最大长边（px）

\- Kimi 识别超时（秒）、DeepSeek 解题超时（秒）、重试次数



\*\*存储位置\*\*：`data/users/{username}\_config.json`（字段由你设计，够用即可）。



\*\*后端逻辑\*\*：启动时读取 `.env` 作为 fallback，用户请求时优先读用户配置。每日花费计数器存内存，凌晨清零。超上限返回 429。



\### 3.2 用户认证

\- Cookie Session，登录后设置 `session\_id`。用户数据存 `data/users/{username}.json`（字段由你设计）。不要用 URL 参数传用户名。



\### 3.3 题目录入

\- 单图上传 / PDF 批量导入。上传 → 保存原图 → 压缩 → base64 → Kimi Vision 识别 → 前端弹窗确认 → 入库。

\- 六级路径：学科 / 考试 / 资料 / 学校 / 大题 / 小题。内部路径用英文映射，用户显示中文。



\### 3.4 智能解题

\- \*\*分流策略\*\*：用户选"几何" → Kimi；选"无几何" → DeepSeek（根据用户配置发 flash 或 pro）；auto → 有图 → Kimi，无图 → DeepSeek。

\- 系统提示词注入：用户画像 + 最近 3 条会话摘要 + 最新每日记忆（<1000 字）。要求：分步讲解、不跳步、公式用 LaTeX。

\- \*\*输出\*\*：SSE 流式输出到前端。



\### 3.5 题库检索

\- 精确指定、范围过滤、关键词模糊搜索、错题筛选、共享题库。



\### 3.6 记忆系统（极简）

\- 静态画像、动态偏好、会话摘要（7 天）、每日压缩。系统提示词只带最近 3 条摘要 + 每日记忆。



\### 3.7 管理模式

\- 路径重命名、题目编辑、自然语言命令改路径、批量操作。



\### 3.8 诊断运维

\- 状态监控、AI 诊断（仅建议，不自动执行）。

\## 四、约束与异常处理（核心要点，不要冗余）



\### 4.1 文件存储

\- \*\*所有 JSON 文件读写必须加 `asyncio.Lock()`\*\*，防止并发覆盖。

\- 写入策略：先写 `.tmp`，再用 `os.replace()` 原子替换。

\- JSON 文件损坏时：备份为 `.bak`，重建空文件，提示用户恢复备份。

\- 所有路径用 `pathlib.Path`，禁止硬编码 `\\` 或 `/`。



\### 4.2 图片上传

\- 校验文件头 Magic Number（JPEG: `FF D8 FF`，PNG: `89 50 4E 47`），不要只看后缀。

\- 压缩参数从前端配置读取（长边、质量），不要写死。base64 后尽量 < 1MB。

\- Kimi 识别 Prompt 必须返回纯 JSON，禁止 Markdown 代码块。如果返回异常，尝试正则提取，失败则提示手动输入。



\### 4.3 依赖安装

\- Pillow 装不上 → 换 OpenCV → 再不行换 imageio → 再不行用标准库简单缩放。不要卡住。



\### 4.4 API 调用

\- Kimi 端点根据用户配置切换（`api.moonshot.cn` 或 `api.kimi.moonshot.cn`）。

\- DeepSeek 根据用户配置发 `flash` 或 `pro`。

\- 超时、重试次数从用户配置读取。默认：Kimi 30s，DeepSeek 60s，重试 2 次，间隔 3s。

\- 降级：DeepSeek 503/429 → 自动切 Kimi；Kimi 不可用 → 返回友好提示。

\- 熔断：内存维护每日花费，超 `daily\_cost\_limit` 返回 429。

\- API Key 从用户配置读取，禁止硬编码。



\### 4.5 前后端联调

\- CORS 允许 `http://localhost:5173`。

\- 前端 `vite.config.js` proxy：`/api` → `http://localhost:6000`。

\- Cookie/Session 鉴权，开发环境 `secure=false, samesite=lax`。

\- 文件上传用 `FormData` + `UploadFile`，确保 `multipart/form-data`。



\### 4.6 流式输出（SSE）

\- 格式 `data: xxx\\n\\n`，结尾 `data: \[DONE]\\n\\n`。前端 `EventSource` 接收，注意断线重连。



\### 4.7 路径操作

\- 重命名时目标已存在 → 提示换名。删除时下面有题 → 提示先删题。非法字符 → 自动替换下划线。



\### 4.8 系统级

\- 磁盘 < 100MB → 提示清理。并发解题过多 → 限制 SSE 连接数（如最多 2 个）。未知异常 → 记录日志，返回错误码，不暴露堆栈。



\### 4.9 跨平台与中文

\- 用户路径显示中文，代码内部映射英文目录名。JSON 统一 UTF-8。

\## 八、一键启动命令（Windows 开发）



项目根目录 `start.bat`：

```bat

@echo off

echo Starting StudyBuddy...

start cmd /k "cd backend \&\& venv\\Scripts\\activate \&\& uvicorn main:app --reload --port 6000"

timeout /t 3 >nul

start cmd /k "cd frontend \&\& npm run dev"

echo.

echo Backend: http://localhost:6000

echo Frontend: http://localhost:5173

echo.

pause

```



\*\*首次启动前\*\*：

```bash

cd backend

python -m venv venv

venv\\Scripts\\activate

pip install -r requirements.txt



cd frontend

npm install

```

\## 十二、特别提醒



1\. \*\*项目比较复杂\*\*：不要只写三五个文件。后端按功能拆成多个 py 文件，前端按页面和组件拆分。

2\. \*\*JSON 格式由你设计\*\*：题目、用户、记忆等 JSON 文件的字段由你按需设计，够用即可，不要冗余。

3\. \*\*每批代码生成后\*\*：列出关键文件、核心函数、以及容易踩坑的地方。

4\. \*\*文件锁\*\*：没有锁 = 并发必翻车。

5\. \*\*Kimi 识别 Prompt\*\*：务必纯 JSON 输出。

6\. \*\*路径映射\*\*：中文显示名和英文目录名的映射表要维护好。

7\. \*\*依赖安装\*\*：Pillow 失败换 OpenCV 或 imageio，不要硬磕。

8\. \*\*一键启动\*\*：`start.bat` 必须能双击运行。

9\. \*\*配置是核心\*\*：用户第一次使用必须先配置 API Key 和每日上限，否则无法解题。

10\. \*\*异常处理\*\*：参考第四章约束，每个异常都要有降级方案，但不要过度设计。
